# Website Architecture & Technical Stack Skill

## Purpose
Define the technical foundation, code organization, deployment strategy, and performance standards for Jaydev's premium Next.js website.

## Tech Stack

### Framework & Runtime
- **Next.js**: 15.5.19 (App Router, SSG/SSR, TypeScript)
- **React**: 18.3.0 (Component-based, hooks)
- **TypeScript**: Full type safety across app
- **Node.js**: 20.x LTS (runtime for build, deployment)

### Styling & Design
- **Tailwind CSS**: 3.4.5 (utility-first, custom theme)
- **Framer Motion**: 11.0.0 (animations, scroll effects, transitions)
- **CSS**: Custom CSS for advanced animations, gradients

### Utilities & Libraries
- **React Intersection Observer**: Scroll reveal animations
- **Next/Image**: Automatic image optimization (WebP, lazy-load)
- **Next/Link**: Internal navigation (client-side prefetch)

### Deployment & Hosting
- **Vercel** (Recommended): Native Next.js support, auto-deployment from GitHub
- **Alternative**: AWS S3 + CloudFront, Netlify, DigitalOcean
- **Database**: Not required (static content); optional CMS for blogs

### SEO & Analytics
- **next-sitemap**: Generate XML sitemap
- **Google Search Console**: Monitoring
- **Google Analytics 4**: User tracking (GA4)
- **Vercel Analytics**: Performance monitoring

## Data Flow & Patterns

### Product Detail Page (Dynamic Route)

**File**: `app/products/[slug]/page.tsx`

```tsx
import { products } from "@/lib/content";

export async function generateStaticParams() {
  // Pre-render all product detail pages at build time
  return products.map((p) => ({ slug: p.id }));
}

export const metadata = ({
  params,
}: {
  params: { slug: string };
}) => {
  const product = products.find((p) => p.id === params.slug);
  return {
    title: `${product?.name} | Jaydev Multicomm`,
    description: product?.description,
  };
};

export default function ProductDetailPage({
  params,
}: {
  params: { slug: string };
}) {
  const product = products.find((p) => p.id === params.slug);
  if (!product) return <div>Product not found</div>;

  return (
    <div>
      <h1>{product.name}</h1>
      <img src={product.image} alt={product.name} />
      <p>Manufacturer: {product.manufacturer}</p>
      {/* Product specs, applications, CTA */}
    </div>
  );
}
```

**Why**: Static generation (SSG) pre-renders pages at build time → ultra-fast delivery.

## Performance Optimization

### Image Optimization

**Problem**: Unoptimized images = slow site.

**Solution**:
```tsx
import Image from "next/image";

export default function ProductCard({ product }) {
  return (
    <Image
      src={product.image}
      alt={product.name}
      width={400}
      height={300}
      placeholder="blur"        // Blur-up effect while loading
      priority={false}          // Lazy-load by default
    />
  );
}
```

**Result**: Automatic WebP conversion, responsive sizing, lazy-loading.

### Code Splitting

**Problem**: All code in one bundle = slow initial load.

**Solution**: Next.js automatically splits by route. Each page loads only its code.

```bash
# Build output shows chunk sizes
.next/static/chunks/
  ├── main-xxxxx.js (framework code)
  ├── app/page-xxxxx.js (homepage code only)
  ├── app/products/page-xxxxx.js (products page only)
```

### Font Loading

```tsx
// app/layout.tsx
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  display: "swap",  // Avoid layout shift during load
});

export default function RootLayout({ children }) {
  return (
    <html className={inter.className}>
      {children}
    </html>
  );
}
```

**Result**: Fonts load asynchronously; page renders text immediately.

### CSS Optimization

Tailwind CSS purges unused utilities at build time:
```bash
# Before build: 3.5 MB CSS (all utilities)
# After build: 45 KB CSS (only used classes)
```

## Troubleshooting

### Build Fails
**Check**: `npm run build` errors. Ensure all imports exist, TypeScript types correct.

### Deploy Fails
**Check**: Environment variables set in Vercel dashboard. File size limits (Vercel max 4.5MB per function).

### Slow Load
**Check**: Image sizes, font loading, lazy-loading enabled, cache headers set.
