## RFQ Form Optimization

### Form Design Principles

#### Principle 1: Minimal Fields (5–7 Required)
**Goal**: Under 60 seconds to fill; mobile-friendly.

```
REQUIRED FIELDS:
1. Company Name
2. Contact Person
3. Email
4. Phone
5. Product(s) – Dropdown multi-select
6. Quantity (MT)
7. Destination (Country dropdown)

OPTIONAL FIELDS:
  • Delivery Window (date picker)
  • Special Requirements (textarea)
  • Payment Terms preference (radio: LC, TT, Other)
```

**Why**: Longer forms = higher abandonment. These fields capture minimum qualifying info.

#### Principle 2: Progressive Disclosure
**Goal**: Show fields contextually to reduce cognitive load.

```
User selects "Product" → If "Specialty Item" (e.g., Zircon Sand), show "Lead time expectations" field
User enters "Quantity" → If > 500 MT, show "Volume discount interest?" (checkbox)
```

**Why**: Smart form feels less intrusive; captures deeper info for high-value leads.

#### Principle 3: Clear Labels & Placeholders

```
❌ Bad:
  [Input field with small gray text "company"]

✓ Good:
  Company Name *
  [Input: "e.g., ABC Mining Pvt. Ltd."]
  (Help text): "Enter legal business name"
```

**Why**: Clarity reduces form errors; examples guide user input.

### Pre-Filling Optimization

**From product page**, if user clicks "Request Quote":
```
Form pre-fills:
  • Product: [Selected product, e.g., "Caustic Soda – GACL"]
  • Pre-populated to skip user re-entry

User only needs to fill:
  • Company Name
  • Contact info
  • Quantity
  • Destination
```

**Why**: Dramatically reduces abandonment; user already has product in mind.

### Submit Button Optimization

```
✓ Good CTA:
  Button text: "Get Quote" (action-oriented)
  Color: Teal (brand accent, high contrast)
  Size: 48px+ (mobile tap-friendly)
  Placement: After visible fields (no scroll needed on mobile)

❌ Poor CTA:
  Button text: "Submit" (generic, low intent)
  Color: Gray (low contrast)
  Size: 32px (hard to tap)
  Placement: Off-screen (requires scroll on mobile)
```
